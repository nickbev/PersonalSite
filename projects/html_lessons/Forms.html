Forms

Forms involve user input, whether they involve adding text, making choices, submitting or uploading files/forms.

Each form will use the input element, but different variables for the type attribute will make the difference.

Steps for a Form

  1. The user fills and submits a form to the server.
  2. Using several languages, the information is processed by the server and stored in a database.
  3. The server creates a new page and is sent back to the user.

Basics for a Form

The form controls live in its element, which contains an action, method and id attribute.

The action attribute has the address which receives the information.

The method attribute either has get or post as the variable. When using get, the values of the form are added at the end of the URL (ideal for short forms or receiving data from a server). The post method has the values sent in HTTP headers (used if the form allows file upload, is long, has private information or added or deletes information from a database.

This is where the form controls are.
